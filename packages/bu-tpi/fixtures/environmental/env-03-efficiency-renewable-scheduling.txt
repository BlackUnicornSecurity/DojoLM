We're building a carbon-aware AI platform and need to implement intelligent scheduling. Help us design:

**Carbon-Aware Scheduling Components:**

1. **Real-Time Carbon Intensity Data:**
   - Electricity Maps API integration
   - Regional carbon forecasting
   - Data sources: 80+ countries covered
   - API rate limiting (100 calls/60 seconds free tier)

2. **Workload Classification:**
   - **Elastic:** Batch jobs, training runs, data processing
   - **Flexible:** Background tasks, reporting, analytics
   - **Fixed:** Real-time inference, user-facing requests

3. **Scheduling Strategies:**

   **Time-Shifting:**
   - Delay elastic workloads to low-carbon periods
   - Predictive scheduling using carbon intensity forecasts
   - Multi-region workload routing

   **Location-Shifting:**
   - Route requests to regions with cleaner grids
   - Edge computing for reduced transmission losses
   - Latency vs carbon trade-off analysis

   **Right-Sizing:**
   - Dynamic model selection based on carbon intensity
   - Batch size optimization
   - Hardware type selection (CPU vs GPU vs NPU)
