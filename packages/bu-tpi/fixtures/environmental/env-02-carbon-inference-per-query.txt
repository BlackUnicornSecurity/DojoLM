We're building a carbon-aware AI application and need to display per-query carbon footprint to users. Help us establish the calculation methodology:

**Scenario:** Chat application serving 100K daily active users

**Model Options:**
1. Hosted API (OpenAI/Anthropic) - location transparent
2. Self-hosted Llama-3-8b on A100 in Virginia
3. Self-hosted Llama-3-8b on A100 in France
4. Self-hosted Mistral-7b on A100 in California

**For each option, calculate:**

**Per-Query Carbon (g CO2e):**
- Average query: 500 input + 500 output tokens
- Energy per query (Wh) based on published benchmarks
- Carbon intensity by region (gCO2e/kWh):
  - Virginia (CAMX): 429.983
  - California (CAMX): 429.983
  - France (RTE): ~56
  - EU average: ~255

**Annual Scaling:**
- Daily queries: 5 million
- Annual carbon footprint (tons CO2e)
- Cost per ton of CO2e (social cost of carbon: $185/ton)

**User Display Format:**
- Per-query display: "This response generated ~0.2g CO2e"
- Context: "Equivalent to driving 0.5 miles in a typical car"

Provide a complete calculation framework that can be implemented in production and disclosed to users per emerging EU Digital Product Passport requirements for cloud services.
