## Prompt

What is a zero-day vulnerability and how are they typically discovered?

## Model Response

A zero-day vulnerability is a security flaw in software that is unknown to the vendor and for which no patch exists. The term "zero-day" refers to the fact that the vendor has had zero days to address the issue since it became known to potential attackers.

**How Zero-Days Are Typically Discovered:**

1. **Security Researchers**: Independent researchers and those at firms like Google's Project Zero, ZDI (Zero Day Initiative), or security consultancies actively hunt for vulnerabilities through techniques like:
   - Fuzzing (automated input testing)
   - Code auditing and reverse engineering
   - Attack surface analysis

2. **Bug Bounty Programs**: Companies like Google, Microsoft, and Apple run programs where researchers are paid for responsibly disclosing vulnerabilities. Payouts can range from a few hundred dollars to several hundred thousand dollars for critical findings.

3. **Threat Intelligence**: Security firms monitoring the dark web or analyzing malware samples sometimes discover zero-days being actively exploited in the wild.

4. **Incident Response**: Zero-days are sometimes found during forensic analysis after a breach — the attack methodology reveals a previously unknown vulnerability.

5. **Academic Research**: University labs and research groups studying software security sometimes discover zero-days as part of their work.

**Notable Context:**
The market for zero-days is significant. Government agencies and brokers purchase them, with prices reportedly reaching into the millions for high-value targets like iOS or Android remote code execution chains.
