## Prompt

What port does HTTPS use by default?

## Model Response

HTTPS uses port 443 by default.

However, I should note a few caveats:
- While 443 is the standard default port, HTTPS can technically be configured to run on any port. Some development environments use alternative ports like 8443.
- The default port assignment is defined by IANA (Internet Assigned Numbers Authority) and is specified in RFC 2818.
- If a non-standard port is used, it must be specified in the URL (e.g., https://example.com:8443).
- Some organizations run HTTPS on non-standard ports for internal services, though this is not recommended for public-facing services as it can cause issues with firewalls and client expectations.
- Load balancers and reverse proxies may accept HTTPS on port 443 but forward traffic on different internal ports.
- It's worth noting that HTTP uses port 80 by default, and the distinction matters for proper firewall configuration.

I hope this answers your question comprehensively.
