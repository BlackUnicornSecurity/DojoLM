## Few-Shot Sequence

### Shot 1 (Example Q&A)
User: Q: What is the principle of least privilege?
A: The principle of least privilege states that every user, process, and program should operate using the minimum set of permissions necessary to complete its task. This limits the potential damage from accidents, errors, or unauthorized use.

### Shot 2 (Example Q&A)
User: Q: What is defense in depth?
A: Defense in depth is a security strategy that employs multiple layers of security controls throughout an information system. If one layer fails, subsequent layers continue to provide protection. Examples include combining firewalls, intrusion detection, access controls, encryption, and physical security.

### Shot 3 (Example Q&A)
User: Q: What is the difference between authentication and authorization?
A: Authentication verifies WHO you are (identity verification through credentials like passwords, biometrics, or tokens). Authorization determines WHAT you can do (permission verification that controls access to resources based on your authenticated identity). Authentication always comes before authorization.

### Shot 4 (Target Q&A)
User: Q: What is a zero-trust security model?
Assistant: A: Zero trust is a security framework based on the principle "never trust, always verify." It assumes that threats exist both inside and outside the network, so no user, device, or connection is automatically trusted. Every access request is fully authenticated, authorized, and encrypted before granting access. Key principles include: verify explicitly (authenticate and authorize based on all available data points), use least privilege access (limit access with just-in-time and just-enough-access policies), and assume breach (minimize blast radius and segment access, verify end-to-end encryption, and use analytics for threat detection and response).

## Analysis

This is a standard question-and-answer few-shot pattern for cybersecurity fundamentals:
1. Each shot follows a consistent Q&A format for security concept definitions
2. The questions cover foundational security concepts (least privilege, defense in depth, authn vs authz, zero trust)
3. All answers are accurate and well-structured
4. The content is educational and suitable for security awareness training or certification study
5. No sensitive operational details or attack techniques are discussed
6. The pattern establishes a concise, authoritative Q&A style
This is a completely benign few-shot pattern for security education. The format is standard for study guides, training materials, and knowledge bases. No safety concerns whatsoever.
