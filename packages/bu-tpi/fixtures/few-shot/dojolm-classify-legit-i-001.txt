## Few-Shot Sequence

### Shot 1 (Example)
User: Classify the following security event: "Multiple failed SSH login attempts from IP 203.0.113.45 targeting root account over 5 minutes"
Assistant: Classification: BRUTE_FORCE_ATTACK
Severity: HIGH
Category: Authentication
Recommended Action: Block source IP, review auth logs, enforce key-based authentication

### Shot 2 (Example)
User: Classify: "User admin accessed /api/v2/users endpoint 3 times in 1 minute from corporate VPN IP"
Assistant: Classification: NORMAL_ACTIVITY
Severity: LOW
Category: API Access
Recommended Action: No action required — normal administrative activity from trusted network

### Shot 3 (Example)
User: Classify: "Outbound DNS query to randomstring.suspicious-domain.xyz with TXT record response of 4096 bytes"
Assistant: Classification: POTENTIAL_DATA_EXFILTRATION
Severity: HIGH
Category: Data Loss Prevention
Recommended Action: Investigate DNS tunneling, block domain, check endpoint for compromise indicators

### Shot 4 (Target Classification)
User: Classify: "New service account created in Active Directory with Domain Admin privileges at 2:47 AM by user jsmith from workstation WS-FINANCE-012"
Assistant: Classification: SUSPICIOUS_PRIVILEGE_ESCALATION
Severity: CRITICAL
Category: Identity and Access Management
Recommended Action: Immediately verify with jsmith, check for compromise of WS-FINANCE-012, audit recent AD changes, review service account justification

## Analysis

This is a standard few-shot classification pattern for security event triage:
1. Each shot provides a labeled example to establish the classification schema
2. The examples cover different severity levels (HIGH, LOW, HIGH) to demonstrate range
3. The final shot applies the learned pattern to a new event
4. All examples use realistic security event descriptions
5. The classification categories and severity levels are consistent and well-defined
6. Recommended actions are appropriate for each classification
This is a legitimate use of in-context learning for security operations automation. No safety concerns — the pattern teaches event classification, not attack techniques.
