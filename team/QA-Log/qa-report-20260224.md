## Bug Reports

### Bug #001: Missing `value` prop on Select.Item component (KNOWN FROM PREVIOUS QA)

**Severity:** Medium
**Environment:** Chrome, Desktop (1920x1080), http://localhost:3000
**Story:** QA-010 (Run Tests tab)
**Frequency:** Always (reproducible every time)
**Status:** KNOWN - Already documented in lessonslearned.md

#### Steps to Reproduce
1. Navigate to http://localhost:3000
2. Click on "Run Tests" tab
3. Open browser DevTools Console (F12)
4. Observe console error

#### Expected Behavior
No console errors should be present when loading any tab.

#### Actual Behavior
Console shows the following error:
```
Error: A <Select.Item /> must have a value prop that is not an empty string.
This is because the Select value can be set to an empty string to clear the selection and show the placeholder.
```

#### Additional Notes
- The UI still renders correctly despite the error
- This is a React component validation error from Radix UI
- **Already fixed according to lessonslearned.md** - but fix may not be deployed
- File to fix: `packages/dojolm-web/src/components/tests/TestRunner.tsx`

### Bug #002: Missing Reset Filters Button

**Severity:** Low
**Environment:** Chrome, Desktop (1920x1080), http://localhost:3000
**Story:** QA-003 (Live Scanner - Engine Filters)
**Frequency:** Always

#### Steps to Reproduce
1. Navigate to Live Scanner tab
2. Uncheck one or more filters
3. Look for a "Reset Filters" button

#### Expected Behavior
A "Reset Filters" button should be available to quickly restore all filters to checked state.

#### Actual Behavior
No "Reset Filters" button exists. Users must manually re-check each filter.

#### Additional Notes
- Low severity as users can manually reset filters
- Would improve UX for quick filter reset
